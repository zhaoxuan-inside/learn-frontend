# storage 对象
允许在浏览器中存储键值对数据

# 核心对象

## localStorage
**生命周期**：永久，除非手动清除
**作用域**：同源（协议+域名+端口）所有标签页/窗口共享
**典型用途**：长期偏好设置、离线数据

## sessionStorage
**生命周期**：当前会话（标签页关闭即清除）
**作用域**：同源且同一个标签页（不同标签页不共享）
**典型用途**：临时表单数据、一次性的交互状态

# API
localStorage 和 sessionStorage 的 API 相同

## setItem("[key]", [value])
## getItem("[key]")
## removeItem("[key]")
## clear()
## key([n])

## length

# 事件 storage
| 存储类型 | 谁来产生事件 | 谁来接收事件
|------|------|----------|
|localStorage	| 当前页面 |其他任何同源页签/窗口
|sessionStorage	| 当前页面 |同一个页签内的其他同源页面（如同源的 \<iframe\>）

# 重要特性与限制
| 特性 | 说明 |
|------|------|
| 存储类型 | 只能存字符串。存对象/数组需 JSON.stringify/parse |
| 同步阻塞 | 所有操作都是**同步**的，大数据量可能阻塞主线程 |
| 容量限制 | 各浏览器不同，一般约 5-10MB（比 Cookie 的 4KB 大很多） |
| 安全性 | 不要存敏感信息（如 token、密码），因为可通过 JS 轻松读取 |
| 跨域限制 | 严格遵循同源策略（协议、域名、端口必须一致） |
| 隐私模式 | Safari 无痕模式下 localStorage 可能完全不可用或抛出异常 |
